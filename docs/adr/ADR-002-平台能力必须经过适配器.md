# ADR-002：平台能力必须经过适配器

- 状态：Accepted
- 日期：2026-07-18

## 背景

相机扫码、工业扫描头、网络、存储、振动、设备信息和打印能力在 H5、通用 Android 与厂商 PDA 上实现不同。

## 决策

业务页面不得直接调用 uni-app 平台 API 或厂商 SDK。所有平台能力通过 `src/platform` 下的接口和适配器提供。

## 适配范围

- Scanner Adapter。
- Network Adapter。
- Storage Adapter。
- Feedback Adapter。
- Device Info Adapter。
- Printer Adapter。

## 候选方案

### 页面直接调用平台 API

实现快，但设备差异扩散到所有页面，无法测试和替换。

### 使用平台适配器

增加接口设计成本，但能隔离 H5、通用 App 和真实 PDA。

## 理由

- 厂商 SDK 接入不应重写业务流程。
- H5 测试需要 Fake Adapter。
- 页面应关注现场任务，而不是设备细节。

## 后果

- 平台接口需要稳定设计。
- Adapter 需处理权限、初始化和释放。
- 页面代码更容易测试。

## 约束

- 页面中禁止出现未封装的 `uni.scanCode` 和存储 API。
- Adapter 错误应转换为统一平台错误。
- 页面离开时释放连续扫描监听器。

## 验证

- H5 可使用 Fake Scanner。
- Android 可使用通用 Scanner。
- 厂商 SDK 替换时页面主流程不变。
