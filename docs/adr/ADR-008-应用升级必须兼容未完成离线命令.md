# ADR-008：应用升级必须兼容未完成离线命令

- 状态：Accepted
- 日期：2026-07-18

## 背景

设备升级时可能仍有待同步、失败、冲突或结果未知命令。若新版本无法读取旧数据，现场操作可能丢失或无法审计。

## 决策

所有持久化命令和关键本地数据必须携带 Schema 版本。应用升级前后要支持兼容读取、迁移或显式人工处理，禁止静默清空未完成命令。

## 候选方案

1. 升级时清空本地数据。
2. 永久维持一个数据格式。
3. Schema 版本与迁移机制。

选择方案 3。

## 理由

- 保证未完成现场操作可恢复。
- 支持命令模型演进。
- 回滚和灰度更安全。

## 后果

- 需要迁移测试和版本治理。
- 回滚必须考虑新旧格式兼容。
- 部分无法自动迁移的数据需转人工。

## 约束

- 发布说明必须标记数据格式变化。
- 升级测试必须包含未完成命令。
- 迁移失败不得删除原始记录。
- 回滚前确认旧版本可读取或提供转换。

## 验证

- 使用旧版本创建命令后升级，新版本可读取。
- 冲突和结果未知状态可迁移。
- 迁移失败提供诊断和人工恢复。
- 回滚不会导致队列不可见。
