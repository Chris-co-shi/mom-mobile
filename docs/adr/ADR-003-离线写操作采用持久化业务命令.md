# ADR-003：离线写操作采用持久化业务命令

- 状态：Accepted
- 日期：2026-07-18

## 背景

工业现场网络可能中断。简单缓存 HTTP 请求或只在内存中保存表单，无法保证应用重启后的恢复、幂等和冲突处理。

## 决策

允许离线的写操作保存为持久化业务命令，而不是任意 HTTP 响应缓存。命令必须包含业务类型、Schema 版本、幂等键、correlation ID、创建时间、尝试次数和显式状态。

服务端幂等仍然是强制要求。

## 候选方案

1. 断网时禁止全部写操作。
2. 缓存原始 HTTP 请求。
3. 持久化业务命令。

选择方案 3。

## 理由

- 命令能表达业务意图。
- 可以进行版本迁移和冲突处理。
- 可以保持固定幂等键。
- 应用重启后仍可恢复。

## 后果

- 需要状态机和同步 Worker。
- 需要本地 Schema 版本。
- 需要定义每类命令的离线资格。
- 用户必须能看到失败和冲突。

## 约束

- Token 不进入命令载荷。
- 同一命令重试保持幂等键。
- 网络恢复后不盲目并发重放。
- 高风险命令可禁止离线。

## 验证

- 断网入队后重启不丢失。
- 同一命令同步三次只执行一次业务。
- 409 进入冲突。
- 超时进入结果未知。
