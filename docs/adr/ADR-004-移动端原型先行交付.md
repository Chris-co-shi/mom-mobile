# ADR-004：移动端业务页面采用原型先行交付

- 状态：Accepted
- 日期：2026-07-18

## 背景

PDA 页面受小屏、扫码、手套、弱网和设备错误影响。直接根据接口字段开发容易遗漏离线、冲突和恢复状态。

## 决策

任何正式 PDA 功能进入实现前，必须完成：

- 用户流程。
- 竖屏原型。
- 页面状态矩阵。
- 组件映射。
- API 映射。
- 权限和工厂范围。
- 幂等和离线规则。
- 硬件交互说明。

## 理由

- 现场交互问题应在编码前暴露。
- 原型能同时评审正常与异常路径。
- 避免把管理后台页面缩小后直接搬到 PDA。

## 后果

- 页面开始编码前增加设计投入。
- 原型和文档需要版本管理。
- 返工风险降低。

## 约束

- 原型图片进入 Git。
- Offline、Conflict、Unknown Result、Hardware Error 和 Manual Takeover 按适用范围强制覆盖。
- 原型不等于页面已经实现。

## 验证

PR 实现业务页面时必须链接对应流程、原型、状态矩阵和 API 映射。
