# ADR-006：离线冲突必须显式处理

- 状态：Accepted
- 日期：2026-07-18

## 背景

离线期间服务端对象可能发生变化，例如送货单已完成、容器被占用、批次被锁定或工单已关闭。自动覆盖会造成库存和生产错误。

## 决策

服务端返回业务冲突时，命令进入 `CONFLICT` 或 `MANUAL_REQUIRED`，移动端展示本地命令摘要、服务端当前状态、冲突原因和允许动作。

禁止使用“最后写入覆盖”作为通用策略。

## 允许动作

- 刷新服务端状态。
- 修正本地输入并使用受控的新业务意图。
- 取消本地命令。
- 转主管或人工处理。

## 理由

- 工业业务状态不能静默覆盖。
- 操作员需要知道命令未完成。
- 冲突处理需要审计。

## 后果

- 后端需要返回结构化冲突信息。
- 移动端需要冲突对比页面。
- 部分命令不能完全自动同步。

## 约束

- 409 不进入自动无限重试。
- 修正后是否复用幂等键需按业务语义决定；原命令不能被静默改写。
- 主管操作需重新权限校验。

## 验证

- 容器占用、状态变化和权限变化可进入冲突页。
- 用户可看到本地和服务端差异。
- 冲突不会自动标记为完成。
