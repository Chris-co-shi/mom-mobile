# ADR-009：P1.5 Mobile 认证运行时

- 状态：Accepted
- 日期：2026-07-19
- 关联文档：[P1.5 Mobile 认证与授权运行时基线](../architecture/P1.5-Mobile认证授权运行时基线.md)
- 后端权威：[mom-platform P1.5 S00 PR #15](https://github.com/Chris-co-shi/mom-platform/pull/15)

## 1. 背景

MOM Mobile 面向 Android 工业 PDA，必须在 App 进程回收、前后台切换、弱网、设备丢失和离线命令场景中安全维护用户授权 Session。旧文档只提出“平台安全存储或最小化持久化”，没有冻结浏览器类型、回调方式、Token 存储、Refresh 不确定状态或离线命令完整身份归属。

## 2. 候选方案

### 2.1 嵌入式 WebView 登录

交互可控，但 App 能观察账号密码输入环境，安全边界和标准 OAuth Native App 实践较差，不采用。

### 2.2 系统浏览器 + 自定义 Scheme

实现较容易，但自定义 Scheme 可能被其他 App 抢占，正式环境不作为首选。

### 2.3 系统浏览器 + Authorization Code + PKCE + HTTPS App Link

账号密码只进入 IAM 页面，回调可通过已验证 HTTPS 域名与 App 关联，适合 Public Client 和 Android PDA。

## 3. 决策

采用方案 2.3，并冻结：

1. 唯一 Client 为 `mom-mobile-pda`。
2. 仅允许具备独立 Mobile Access 的 `INTERNAL` 用户。
3. 使用系统浏览器，不在嵌入式 WebView 输入账号密码。
4. 使用 Authorization Code + PKCE S256 + OIDC。
5. 正式环境优先 HTTPS App Link；自定义 Scheme 只作受控后备。
6. Access Token 只存在内存。
7. Refresh Token 通过 Android 安全存储 Adapter 持久化。
8. PKCE 一次性事务可持久化并承受进程回收。
9. 冷启动优先 Refresh 恢复 Session。
10. 同一 App 实例使用 Single Flight Refresh；每业务请求最多自动重试一次；403 不刷新。
11. Refresh Token 本地替换结果不确定时，禁止再次使用旧 Token。
12. H5 按 Web 内存模型处理，不持久化真实 Refresh Token。
13. 离线命令必须绑定用户、`sid`、Client、Factory、Party、Permission 和创建时间。
14. 用户 A 的命令不得由用户 B 自动同步。
15. `X-Factory-Id` 只是工作上下文，不是授权证明。
16. Gateway 不是最终授权边界，业务服务执行最终 Permission、Factory/Party 和对象归属校验。

## 4. 后果

### 正向

- 账号密码不进入 App WebView。
- Access Token 不落盘，Refresh Token 使用平台安全能力。
- App 进程回收后仍能安全完成授权事务。
- 冷启动和弱网 Refresh 有明确的 Fail Safe 行为。
- 离线命令与创建身份、Session 和 Factory 可审计绑定。

### 负向

- 需要 Android App Link 域名关联、证书和真机验证。
- 需要原生安全存储 Adapter 和原子替换语义。
- 进程回收、网络不确定和离线命令恢复测试复杂度高。
- H5 不能替代正式 Android 安全验收。

## 5. 风险与缓解

| 风险 | 缓解 |
|---|---|
| App Link 被错误环境或应用接收 | 精确 Redirect URI、域名关联、环境隔离、state/nonce 校验 |
| Refresh 已在服务端轮换但本地未保存 | 旧 Token 不重试，进入重新登录或受控恢复 |
| 用户切换后同步旧命令 | 命令绑定 user/sid/client/factory/party/permission，同步前校验 |
| H5 误当生产安全实现 | 文档和 CI 明确 H5 只用于开发调试 |
| 设备丢失后 Token 可继续使用 | 服务端 Session 撤销、Mobile Access 关闭、本地安全存储清理 |

## 6. 验证

- 系统浏览器与 App Link 真机测试。
- PKCE、state、nonce、Redirect URI 和进程回收测试。
- Android 安全存储、退出和设备丢失测试。
- 并发 Refresh、网络不确定和旧 Token 不重试测试。
- Client/user_type/Mobile Access 隔离测试。
- 冷启动、前后台和 Session 撤销测试。
- 离线命令跨用户、跨 Session、跨 Factory 和 Permission 变化测试。
- H5 敏感存储扫描。

## 7. 替代条件

只有通过新的跨仓库 ADR 才能评估其他 Native App OAuth 或设备会话方案。不得局部回退为 WebView 密码登录、普通 Storage 保存 Refresh Token、重复使用不确定旧 Token或客户端静默改写离线命令归属。
