# 原料收货用户流程

## 1. 目标

仓库收货员通过扫描供应商送货单和容器，确认重量并建立待检批次；网络不可用时，将符合离线规则的收货命令安全进入队列。

## 2. 角色

- 主要角色：仓库收货员。
- 协作角色：质量交接人员、仓库主管。

## 3. 前置条件

- 用户已登录。
- 已选择工厂和仓库。
- 具有收货权限。
- 送货单已由 ERP 或供应商协同流程建立。
- 设备扫码能力可用，或允许受控手工输入。

## 4. 正常流程

```text
进入待收货任务
→ 扫描供应商送货单
→ Gateway 查询送货信息
→ 展示供应商、物料、采购参考和预计数量
→ 扫描已有容器或生成新容器
→ 录入毛重、皮重或净重
→ 查看提交摘要
→ 生成 idempotencyKey 和 correlationId
→ 提交收货命令
→ 展示收货单号、批次号和标签状态
```

## 5. 扫描规则

### 5.1 送货单

- 扫码来源：二维码、条码或 Data Matrix。
- 允许手工输入：仅在权限允许且扫码设备不可用时。
- 重复扫描：显示当前送货单状态，不重复创建。

### 5.2 容器

- 可扫描既有容器。
- 新容器生成需由后端确认编号规则。
- 已占用、已关闭或不属于当前工厂的容器不得使用。

## 6. 重量输入

- 支持毛重、皮重和净重组合。
- 页面进行基本非负和精度校验。
- 服务端执行单位、容差和业务规则校验。
- 后续称重设备接入通过独立 Adapter，不直接写入页面。

## 7. 离线流程

```text
提交前检测网络
→ 命令符合离线资格
→ 生成固定幂等键和关联 ID
→ 持久化到离线队列
→ 展示本地命令编号和待同步状态
→ 网络恢复后受控同步
→ 展示完成 / 冲突 / 结果未知 / 人工处理
```

离线页面必须明确说明：命令已保存在设备，不代表服务端已经完成收货。

## 8. 异常流程

### 8.1 送货单不存在

- 保留扫描值。
- 允许重新扫描或手工修正。
- 不创建离线写命令。

### 8.2 送货单已完成

- 展示已存在的收货结果。
- 不允许重新提交。

### 8.3 容器冲突

- 展示容器当前状态。
- 允许更换容器或转主管处理。

### 8.4 提交超时

- 进入结果未知。
- 保留原幂等键。
- 查询收货命令最终状态。
- 不自动新建另一条收货命令。

### 8.5 会话过期

- 保存尚未提交的表单草稿。
- 引导重新登录。
- 登录后重新校验权限和送货单状态。

## 9. 完成状态

成功后展示：

- 收货单号。
- 原料批次号。
- 容器号。
- 重量。
- 待检状态。
- 标签打印状态。
- 下一步：继续收货、查看详情或返回任务列表。

## 10. 验收场景

- 正常在线收货。
- 送货单重复扫描。
- 容器无效。
- 重量错误。
- 断网入队并重启恢复。
- 同一命令同步三次只产生一条收货结果。
- 超时后查询到已成功。
- 服务端状态变化产生冲突。
