# 移动端用户流程规范

每个现场作业页面在原型和实现前必须先完成角色级用户流程。

## 1. 推荐表示

```text
角色
→ 任务入口
→ 扫描或选择对象
→ 本地基础校验
→ 服务端校验
→ 用户确认
→ 在线提交或离线入队
→ 结果或恢复
```

## 2. 必填内容

- 业务目标。
- 操作角色。
- 前置条件。
- 当前工厂和仓库范围。
- 正常路径。
- 取消和返回路径。
- 扫码失败路径。
- 网络中断路径。
- 会话过期路径。
- 冲突路径。
- 结果未知路径。
- 人工接管路径。

## 3. 扫码步骤

每一步扫描需说明：

- 扫描对象。
- 码制和格式。
- 是否允许手工输入。
- 重复扫描处理。
- 扫描器错误处理。
- 扫描结果是否需要在线校验。

## 4. 写操作

需说明：

- 命令名称。
- 幂等键生成时机。
- 是否允许离线。
- 离线命令最大有效期。
- 状态查询方式。
- 冲突和人工恢复动作。

## 5. 任务中断

需说明应用发生以下情况时如何恢复：

- 切到后台。
- 应用被系统回收。
- 应用被强杀。
- 设备重启。
- 用户退出。
- 用户切换工厂。
- 应用升级。

## 7. 验收

- 流程没有仅写“失败提示”而无恢复动作的分支。
- 离线和在线路径最终状态清晰。
- 页面状态矩阵能覆盖全部流程节点。
- 原型和 API 映射与流程一致。
