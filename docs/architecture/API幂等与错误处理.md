# API、幂等与错误处理

## 1. Gateway-only

移动端只访问 MOM API Gateway，不直接访问 IAM、MES、WMS、QMS 或其他内部服务。

## 2. 请求上下文

所有请求应统一携带：

- Authorization。
- Factory Context。
- correlation ID。
- App Version。
- Device ID 或匿名设备标识。
- 写命令的 idempotency key。

不得在 URL 中传递 Token。

## 3. 幂等键

- 用户第一次提交命令时生成。
- 在线重试、离线同步和状态查询期间保持不变。
- 同一业务命令不能每次点击都生成新键。
- 新业务意图必须生成新键。

## 4. 错误模型

建议统一映射：

```text
httpStatus
businessCode
message
correlationId
retryable
conflict
unknownResult
fieldErrors
```

页面不直接解析不同服务的原始响应格式。

## 5. 状态码处理

| 状态 | 移动端处理 |
|---|---|
| 400 | 显示参数或业务输入错误，不自动重试 |
| 401 | 保存可恢复页面状态，引导重新登录 |
| 403 | 展示无权限或工厂范围限制 |
| 404 | 提示对象不存在或已失效，允许刷新 |
| 409 | 进入冲突流程，展示服务端当前状态 |
| 422 | 展示业务规则拒绝 |
| 429 | 遵循 Retry-After，不高频重试 |
| 5xx | 区分可重试与不可确认结果 |

## 6. 超时

查询超时可以允许用户重试。

写命令超时不能直接判定失败：

- 保留幂等键。
- 标记结果未知。
- 调用状态查询。
- 无法确认则转人工处理。

## 7. 离线判定

只有以下情况可考虑入队：

- 明确断网。
- 连接建立失败。
- 请求尚未到达服务端且可确定未执行。

请求已发送后超时，不应简单新建离线命令，否则可能重复执行。

## 8. API 映射文档

每个页面必须记录：

- 查询 API。
- 命令 API。
- 权限码。
- 工厂范围。
- 幂等要求。
- 离线资格。
- 业务错误码。
- 状态查询 API。
- 追踪字段。

## 9. 日志

可记录：

- API 名称。
- 状态码。
- 业务码。
- correlation ID。
- 耗时。
- 应用版本。

禁止记录：

- Token。
- 密码。
- 完整身份证明。
- 敏感业务载荷。

## 10. 验收

- 相同幂等键重试三次只产生一个业务结果。
- 401 不丢失可恢复表单。
- 409 进入冲突页而不是普通 Toast。
- 429 不造成请求风暴。
- 写命令超时进入结果未知。
- 错误页面能复制 correlation ID。
