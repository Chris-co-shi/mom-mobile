# 模块边界

## 1. 当前目录

```text
src/
├── api/
├── components/
├── idempotency/
├── offline/
├── pages/
├── platform/
├── stores/
├── App.vue
├── main.ts
├── manifest.json
└── pages.json
```

## 2. 依赖方向

```text
pages
├── components
├── stores
├── api
├── offline
└── platform interfaces

components
└── shared view models

offline
├── api
├── idempotency
└── platform/storage + platform/network

api
├── idempotency
└── session context

platform
└── uni-app API or vendor SDK
```

## 3. Pages

允许：

- 调用扫描接口。
- 调用 API 或提交离线命令。
- 更新页面状态。
- 路由跳转。

禁止：

- 直接使用 `uni.request`。
- 直接使用存储 API。
- 直接调用厂商 SDK。
- 在页面内实现重试定时器。
- 将 Token 写入页面数据或日志。

## 4. Components

- 不依赖具体页面。
- 不访问网络。
- 不写本地存储。
- 通过 Props、Slots 和 Events 协作。
- 展示业务状态时使用标准状态枚举和设计 Token。

## 5. API

- 只暴露 Gateway 契约。
- 不暴露内部服务地址。
- 统一请求上下文和错误模型。
- 业务 API 按领域或垂直切片组织。
- API DTO 不作为页面长期状态模型。

## 6. Offline

- 命令模型与 UI 模型分离。
- Repository 只负责持久化。
- Service 负责编排状态迁移。
- Sync Worker 负责受控同步。
- Conflict Resolver 负责生成差异和人工动作。

## 7. Platform

建议接口：

```text
ScannerAdapter
NetworkAdapter
StorageAdapter
FeedbackAdapter
DeviceInfoAdapter
PrinterAdapter
```

不同平台实现放在适配器内部，页面只依赖接口。

## 8. Stores

Store 不得替代：

- 离线命令仓储。
- 服务端权威数据。
- 本地安全存储。

## 9. 边界校验

CI 应逐步增加：

- 页面禁止直接调用平台 API 的静态检查。
- 页面禁止直接调用 `uni.request`。
- `platform` 不依赖业务页面。
- `components` 不依赖 `pages`。
- 文档规定的关键目录必须存在。
