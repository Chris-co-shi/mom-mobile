# 本地存储与数据安全

- 当前阶段：`P1.5：认证与授权闭环`
- Auth Runtime：[P1.5 Mobile 认证与授权运行时基线](P1.5-Mobile认证授权运行时基线.md)

## 1. 原则

PDA 本地存储只用于当前现场作业、OAuth 一次性事务和离线恢复，不是业务权威数据库。普通业务存储与 Auth 安全存储必须分离。

## 2. 存储分区

### 2.1 Auth Secure Storage

仅存：

- 当前有效 Refresh Token。
- 原子替换所需的最小状态。
- 必要的 Session/Client 本地关联元数据。

要求：

- 使用 Android 平台安全存储能力和专用 Adapter。
- 不通过普通 `uni.setStorage`、明文文件、普通 SQLite 字段或 Pinia 持久化访问。
- 不硬编码加密密钥，不把混淆当作加密。
- 退出、Session 撤销、Mobile Access 关闭和设备丢失时清理。

### 2.2 PKCE Transaction Storage

可短期持久化：

- `state`
- `nonce`
- `code_verifier`
- `returnRoute`
- `client_id`
- Redirect URI 标识
- 创建/过期时间

用于承受系统浏览器期间 App 进程被回收。回调成功、失败、取消或过期后立即清理。

### 2.3 Offline Business Storage

可持久化：

- 离线命令与状态。
- 用户、`sid`、Client、Factory、Party、Permission 归属。
- 幂等键、Correlation ID、Schema 版本、错误摘要和冲突快照。
- 必要页面恢复草稿。
- 非敏感当前 Factory 偏好。

## 3. 禁止持久化

- 密码或密码摘要。
- Access Token。
- ID Token 原文。
- Refresh Token 到普通 Storage。
- Authorization Code。
- 完整 Authorization Header。
- JWT 私钥或 Client Secret。
- Token、`code_verifier` 或敏感载荷到日志、截图、诊断文件。
- 完整业务数据库镜像或不必要的客户/个人敏感数据。

## 4. Access Token 与 Store

- Access Token 只在内存 Auth State 中存在。
- Pinia 可保存内存会话状态，但不得配置持久化 Token。
- App 进程结束后 Access Token 丢失，冷启动使用 Secure Storage 中的 Refresh Token 恢复。

## 5. Refresh Token 原子替换

1. 发起 Refresh 前读取当前安全记录。
2. 服务端成功返回新 Token 后，以原子方式替换本地记录。
3. 新记录确认持久化后，旧记录才视为删除。
4. 网络或本地写入结果不确定时，旧 Token 不得再次使用。
5. 进入重新登录或受控恢复，不实现客户端宽限窗口。

## 6. Key 与隔离

普通业务存储建议：

```text
mom-mobile:{environment}:{user}:{session}:{factory}:{domain}:{key}
```

要求：

- 环境、用户、Session 和 Factory 不共享业务 Key。
- Auth Secure Storage 使用独立、不可由页面拼接的命名空间。
- Party 和 Client 作为命令记录字段保存，不依赖 Key 隐式推断。

## 7. Storage Adapter 边界

页面不得直接使用 `uni.setStorage`、`uni.getStorage` 或原生安全存储 API。

- 普通 Storage Adapter：离线业务数据、Schema、过期、容量和迁移。
- Auth Secure Storage Adapter：Refresh Token 与原子替换。
- PKCE Transaction Adapter：一次性授权事务。

三个 Adapter 的数据类型、权限和清理策略不得混用。

## 8. 生命周期

每类数据必须定义创建、更新、过期、清理、退出、Factory 切换、Session 撤销和升级迁移行为。

- 退出：清理 Token 和 PKCE 事务，不静默删除离线命令。
- Factory 切换：不自动同步原 Factory 命令。
- 用户切换：冻结其他用户命令。
- Session 撤销：停止同步，保留命令归属和状态。
- 应用升级：先迁移再读取，失败时进入人工恢复。

## 9. H5 限制

H5 不具备 Android Auth Secure Storage：

- Token 按 Web 内存模型处理。
- 不持久化真实 Refresh Token。
- 使用非生产环境和开发 Client。
- H5 验证不能替代 Android 真机安全验收。

## 10. 容量、日志与截图

- 监控离线队列数量和占用空间。
- 对日志、图片、快照和临时缓存设置上限。
- 达到阈值时停止新增高风险命令并提示处理。
- 错误日志只记录脱敏摘要和 Correlation ID。
- 禁止自动上传包含 Token、账号或敏感业务数据的截图。

## 11. 验收

- Access Token 不落盘。
- Refresh Token 只存在 Android 安全存储。
- PKCE 事务可承受进程回收并按时清理。
- 不同环境、用户、Session 和 Factory 数据隔离。
- 不确定 Refresh 替换不重复使用旧 Token。
- 退出/撤销后 Token 清理但离线命令不丢失。
- H5 不持久化真实 Refresh Token。
- 仓库和日志中无真实密钥、Token 或签名文件。
