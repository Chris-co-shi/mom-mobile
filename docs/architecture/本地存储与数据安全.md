# 本地存储与数据安全

## 1. 原则

PDA 本地存储只用于支撑当前现场作业和离线恢复，不作为业务权威数据库。

## 2. 数据分类

### 2.1 可持久化

- 离线命令。
- 命令状态与错误摘要。
- 应用版本和设备诊断信息。
- 非敏感的当前工厂配置。
- 必要的页面恢复草稿。

### 2.2 受限持久化

- 用户标识。
- 容器、批次和工单摘要。
- 服务端冲突快照。

这类数据应设置最小字段、过期时间和清理策略。

### 2.3 禁止持久化

- 密码。
- 私钥和签名材料。
- 完整 Access Token 日志。
- 不必要的客户或个人敏感数据。
- 完整业务数据库镜像。

## 3. Storage Adapter

页面不得直接调用 `uni.setStorage`、`uni.getStorage` 等 API。

Storage Adapter 负责：

- Key 命名。
- 序列化。
- Schema 版本。
- 过期时间。
- 容量检查。
- 错误处理。
- 可选加密。

## 4. Key 设计

建议：

```text
mom-mobile:{environment}:{user}:{factory}:{domain}:{key}
```

禁止不同环境、用户和工厂共享同一个业务 Key。

## 5. 数据加密

需要加密的数据应使用平台安全存储或经过评审的加密方案。

不得：

- 将加密密钥硬编码在前端包中。
- 使用固定明文密钥。
- 把混淆当作加密。

## 6. 生命周期

数据应定义：

- 创建时间。
- 更新时间。
- 过期时间。
- 清理条件。
- 账号退出处理。
- 工厂切换处理。
- 应用升级迁移。

## 7. 离线命令

离线命令不能在退出登录时直接删除，因为可能导致业务操作丢失。

应按用户和工厂隔离，并在重新登录或主管处理后继续处置。

## 8. 容量管理

- 监控本地队列数量和占用空间。
- 对日志、图片和临时缓存设置上限。
- 队列达到阈值时停止新增高风险离线命令并提示处理。
- 清理已完成命令前保留必要审计窗口。

## 9. 数据迁移

每类持久化数据应有 Schema 版本。

应用升级时：

- 先迁移再读取。
- 迁移失败不得静默丢弃命令。
- 提供导出诊断或人工恢复路径。

## 10. 日志与截图

- 错误日志不记录完整业务载荷。
- 禁止自动上传包含敏感数据的页面截图。
- 用户主动反馈时应提示将上传哪些信息。

## 11. 验收

- 不同用户和工厂的数据隔离。
- 退出后 Token 清理但离线命令不丢失。
- 应用升级后旧命令可迁移。
- 存储容量不足时有明确提示。
- 仓库中无真实密钥和签名文件。
