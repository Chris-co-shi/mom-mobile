# 移动端可观测性

## 1. 目标

移动端诊断需要回答：谁、在哪台设备、哪个应用版本、在哪个页面、执行了什么操作、关联到哪条后端链路，以及本地离线命令处于什么状态。

## 2. 诊断上下文

建议统一记录：

- appVersion。
- buildNumber。
- gitSha。
- environment。
- deviceModel。
- osVersion。
- userId 的脱敏标识。
- factoryId。
- pageRoute。
- correlationId。
- commandId。
- idempotencyKey 的脱敏摘要。
- networkType。

## 3. 请求链路

移动端生成或接收 correlation ID，并通过 Gateway 传递到后端 Trace。

页面错误详情应允许复制 correlation ID，但不暴露 Token 和内部堆栈。

## 4. 离线命令指标

至少关注：

- 待同步数量。
- 失败数量。
- 冲突数量。
- 结果未知数量。
- 最旧待处理时间。
- 同步成功率。
- 平均尝试次数。

## 5. 本地日志

日志级别：

- INFO：应用启动、版本、网络切换、命令状态变化。
- WARN：可恢复错误、重试、队列接近上限。
- ERROR：无法恢复的存储、迁移、设备和同步错误。

禁止记录：

- Token。
- 密码。
- 完整业务载荷。
- 完整个人敏感信息。

## 6. 错误报告

错误报告应包含：

- 用户可理解描述。
- 恢复动作。
- correlation ID。
- 应用和设备版本。
- 页面。
- 最近网络状态。
- 可选的离线命令 ID。

## 7. 设备诊断页

建议展示：

- 应用版本和构建号。
- 设备型号和系统版本。
- 当前环境和 Gateway 地址摘要。
- 当前用户和工厂。
- 网络类型。
- 扫描器能力。
- 离线队列统计。
- 最近错误关联 ID。

敏感字段需要脱敏。

## 8. 崩溃与异常

后续可接入崩溃收集，但必须：

- 评估数据合规。
- 过滤 Token 和业务载荷。
- 记录版本和设备维度。
- 支持关闭或环境隔离。

## 9. 后端关联

排障流程：

```text
PDA 错误页
→ correlation ID
→ Gateway 日志
→ 后端 Trace
→ 业务命令 / Outbox / 库存流水
```

## 10. 验收

- 任一 API 错误可获得 correlation ID。
- 离线命令详情可查看最近错误和尝试次数。
- 日志中无 Token。
- 设备诊断页可显示版本、网络和队列状态。
- 运维可通过关联 ID 找到后端链路。
