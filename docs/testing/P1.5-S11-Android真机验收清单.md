# P1.5 S11 Android 真机验收清单

- 当前自动化边界：H5 构建、Fake/Memory Adapter、PKCE、OIDC、冷启动、Single Flight、Fail Closed 与离线身份隔离。
- 当前未在此 CI 环境验证：Android Keystore、系统浏览器回跳、HTTPS App Link 域名关联和进程回收真机行为。

## 1. 构建与环境

- [ ] Release 包使用生产签名，未包含调试证书、测试 Client、测试 Token 或明文密钥。
- [ ] `VITE_MOM_IAM_ISSUER` 与 `VITE_MOM_MOBILE_REDIRECT_URI` 都是 HTTPS。
- [ ] 唯一 OAuth Client 为 `mom-mobile-pda`，Redirect URI 与 IAM 精确一致。
- [ ] APK/AAB 的 Application ID、签名 SHA-256 与 `assetlinks.json` 完全匹配。

## 2. 系统浏览器与 App Link

- [ ] 登录使用系统浏览器，账号密码不进入 WebView。
- [ ] Android 设置显示 App Link 域名已验证；`adb shell pm get-app-links <package>` 状态成功。
- [ ] 正常回调只唤起目标 App 和目标环境。
- [ ] 错误域名、路径、Client、Redirect URI、state、nonce、过期事务全部拒绝并清理事务。
- [ ] 浏览器登录期间强制结束 App 进程，回调后可使用持久化 PKCE 事务安全完成一次。
- [ ] 同一回调重复打开不能再次换 Token。

## 3. Android 安全存储

- [ ] 原生 Adapter 的 `implementation` 固定为 `android-keystore`。
- [ ] Refresh Token 使用 Android Keystore 支持的密钥加密；密钥不可导出，不硬编码。
- [ ] READY → IN_FLIGHT 与 IN_FLIGHT → 新 READY 使用同一安全记录的原子 CAS。
- [ ] 在发出 Refresh 后、收到响应后、提交替换前后分别强制杀进程；残留 IN_FLIGHT 均要求重新登录，旧 Token 不重用。
- [ ] Adapter 返回 UNKNOWN/REJECTED 时清理凭证并进入重新认证。
- [ ] Access Token、ID Token、Refresh Token 不出现在 SharedPreferences 明文、普通 uni storage、SQLite、文件、日志、截图或崩溃报告。
- [ ] 退出、Session 撤销、Mobile Access 关闭和设备丢失处置会清理安全存储，但不会静默删除离线命令。

## 4. Session 与并发

- [ ] 冷启动先 Refresh，再读取 `/api/iam/me`；无 Refresh Token 时不伪造登录态。
- [ ] 并发 401、前台恢复和手工刷新只发出一个 Refresh 请求。
- [ ] 每个业务请求最多自动重试一次；403 不触发 Refresh。
- [ ] Session 撤销、Refresh 重放、12 小时绝对期限、用户禁用或 Client 禁用均进入重新登录。
- [ ] `/api/iam/me` 的 `sid`、`clientId`、`userType`、`mobileAccessEnabled`、Permission 和 Factory 变化即时生效。

## 5. Factory 与离线队列

- [ ] Factory 只能从 `/api/iam/me.factoryIds` 选择，`X-Factory-Id` 不能绕过服务端授权。
- [ ] Mobile 无 Supplier/Customer Party 切换入口。
- [ ] 登录切换前明确显示未同步命令数量并要求确认。
- [ ] 用户 B、不同 `sid`、不同 Factory、Permission 已撤销或旧 Session 已撤销时，旧命令均不自动发送。
- [ ] 离线记录完整包含 `userId`、`sid`、`clientId`、`factoryId`、`partyType/partyId`、`requiredPermission`、`createdAt`、`idempotencyKey`、`correlationId`。

## 6. 验收证据

- [ ] 保存设备型号、Android 版本、App 版本、包名、签名指纹和测试环境。
- [ ] 保存脱敏后的 `pm get-app-links`、用例结果和失败截图；不得包含 Token、Code 或 Authorization Header。
- [ ] Phase 02 Mobile 正式联调前将真机结果链接回阶段验收记录；不得以 H5 结果替代 Keystore、App Link 或强杀恢复证据。
