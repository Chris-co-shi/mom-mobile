# 移动端测试策略

## 1. 测试目标

验证页面流程、平台适配器、离线状态机、幂等和 Android PDA 行为，而不只验证 H5 能否构建。

## 2. 测试层次

### 2.1 静态检查

- 目录边界。
- TypeScript 类型。
- 页面不得直接调用平台 API。
- 页面不得直接调用 `uni.request`。
- DCloud 版本一致。
- 文档关键路径存在。

### 2.2 单元测试

重点：

- 幂等键生成与复用。
- 命令状态迁移。
- 重试和退避计算。
- 错误映射。
- DTO 到 View Model 映射。
- 扫码值解析。
- 存储 Schema 迁移。

### 2.3 Adapter 测试

使用 Fake Adapter 验证：

- 扫码成功、取消、权限拒绝和设备故障。
- 网络在线、弱网、断网和恢复。
- 存储失败、容量不足和数据损坏。
- 振动和声音反馈调用。

### 2.4 组件测试

- 大触控按钮。
- 网络徽标。
- 扫描触发器。
- 命令状态面板。
- 冲突对比。
- 错误恢复面板。

### 2.5 页面流程测试

- 收货正常流程。
- 上架双扫描。
- 领料分次提交。
- 发运连续扫描。
- 离线入队。
- 会话过期。
- 冲突和结果未知。

### 2.6 H5 E2E

H5 使用 Fake Adapter 验证确定性流程：

- 路由。
- 页面状态。
- API Mock。
- 离线队列 UI。
- 错误恢复。

H5 通过不代表真机验证完成。

### 2.7 Android 真机测试

至少验证：

- 安装、升级和卸载。
- 相机权限。
- 硬件扫描头。
- Wi-Fi/蜂窝切换。
- 切后台和强杀。
- 设备重启。
- 存储容量不足。
- 长时间连续扫描。
- 电量和性能。

## 3. 离线测试矩阵

| 场景 | 预期 |
|---|---|
| 提交前断网 | 允许的命令入队 |
| 提交中断网 | 进入结果未知 |
| 入队后强杀 | 重启后命令仍在 |
| 网络恢复 | 受控同步，不并发重复 |
| 409 | 进入冲突 |
| 429 | 按等待时间重试 |
| Token 过期 | 暂停同步并登录 |
| 切换账号 | 不执行旧账号命令 |
| 升级 App | 旧命令可迁移读取 |

## 4. 幂等测试

- 同一命令重复点击。
- 同一幂等键重复同步三次。
- 应用重启后再次同步。
- 请求超时后查询最终状态。
- 手工重试保持原幂等键。

## 5. 视觉和可用性

- 常见 PDA 竖屏尺寸。
- 大字号和长文本。
- 手套操作触控面积。
- 强光和暗色环境可读性。
- 状态不只依赖颜色。
- 系统字体缩放。

## 6. 性能

- 首屏和页面切换。
- 连续扫描响应。
- 数百条离线命令列表。
- 本地存储读取和迁移。
- 长时间运行内存。

## 7. 安全测试

- 日志中无 Token。
- 退出后会话清理。
- 不同用户/工厂数据隔离。
- 安装包无开发密钥。
- 签名和发布渠道正确。

## 8. CI 门禁

当前：

```bash
pnpm install --frozen-lockfile
pnpm validate
pnpm type-check
pnpm build:h5
```

后续逐步增加：

- 单元测试。
- 组件测试。
- H5 E2E。
- 文档链接检查。
- 依赖许可证和漏洞检查。

## 9. 发布门禁

- CI 通过。
- 真机核心流程通过。
- 离线与升级兼容测试通过。
- 无 P0/P1 未解决缺陷。
- 签名和版本信息正确。
- 回滚包可用。
