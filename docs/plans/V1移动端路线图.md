# V1 移动端路线图

## 1. 总体目标

在 V1 中完成一套可演示、可离线恢复、可在 Android PDA 上逐步落地的工业现场作业客户端。

## 2. 阶段划分

| 阶段 | 目标 | 核心输出 |
|---|---|---|
| Mobile Phase 01 | 技术骨架 | App Shell、平台适配器、API Client、离线命令基础 |
| P1.5 | 认证授权闭环 | **Completed / Merged** |
| Mobile Phase 02 | 入厂与上架 | **Pending / Ready after preflight cleanup**；收货、检验交接、上架页面与接口闭环 |
| Mobile Phase 03 | 生产与出库 | 领退料、成品入库、发运、自动同步和冲突处理 |
| Mobile Phase 04 | 真机与演练 | 厂商 SDK、签名发布、弱网、升级和故障演练 |

## 3. Phase 01

目标：建立可以安全承载正式业务页面的移动端基础。

输出：

- uni-app Vue 3 应用骨架。
- H5 构建和 CI。
- 页面路由与工作台。
- Scanner、Network、Storage Adapter。
- Gateway API Client。
- 幂等键和 correlation ID。
- 持久化离线命令基础。
- 原型、状态矩阵和 API 映射模板。

## 4. Phase 02

目标：完成原料进入工厂后的 PDA 作业闭环。

输出：

- 收货任务列表。
- 送货单扫描。
- 容器和重量确认。
- 待检批次建立。
- 检验交接。
- 容器与库位双扫描上架。
- 离线提交和恢复验证。

## 5. Phase 03

目标：覆盖生产物流和成品仓储。

输出：

- 工单领料。
- 生产退料。
- 成品入库。
- 发运装车确认。
- 自动同步 Worker。
- 退避、冲突和人工处理。
- 命令最终状态查询。

## 6. Phase 04

目标：在真实 PDA 和生产模拟环境中完成工程验证。

输出：

- 选定 PDA 型号与扫描 SDK。
- 真机扫描和权限适配。
- Android 签名、包版本和发布流程。
- Wi-Fi 切换、断网、网络抖动演练。
- 应用强杀与重启恢复。
- 升级和回滚下离线队列兼容性验证。
- 设备日志和关联 ID 诊断。

## 7. 阶段门禁

进入下一阶段前必须满足：

- CI 通过。
- 文档与实现一致。
- 关键页面完成原型与状态矩阵。
- 写操作具备幂等策略。
- 离线操作明确可执行与不可执行范围。
- 错误和冲突具备恢复路径。

## 8. V1 完成定义

- Android PDA 上可演示核心现场流程。
- H5 CI 可重复构建。
- 断网后命令不丢失。
- 同一命令重复同步不重复执行业务。
- 冲突不会静默覆盖服务端状态。
- 应用重启后可恢复未完成任务。
- 设备和请求问题可通过版本、页面和 correlation ID 定位。
