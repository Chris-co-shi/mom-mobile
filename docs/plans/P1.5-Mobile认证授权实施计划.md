## 1. 仓库职责

`mom-mobile` 只负责 Android PDA 运行时：系统浏览器 PKCE、HTTPS App Link、Token 安全存储、冷启动/前后台恢复、Single Flight Refresh、Mobile Access 体验控制和离线命令身份归属。

本仓库不定义后端用户、Token、Session、RBAC、Factory/Party Scope、Gateway 或业务服务最终授权语义。

## 2. 当前真实状态

S11 已在 `phase/p1.5-auth-authorization` 阶段分支完成：

- uni-app Vue 3 应用和 H5 构建骨架。
- 首页、收货、上架、领料、发运和离线页面骨架。
- Scanner、Network、Storage Adapter。
- 固定 `mom-mobile-pda` 的系统浏览器 Authorization Code + PKCE S256 + OIDC。
- 可承受进程回收的一次性 PKCE 事务与精确 App Link/state/nonce 校验。
- Access Token 仅内存；H5 Refresh Token 仅内存且不持久化。
- Android Keystore 安全存储 Adapter Contract，READY/IN_FLIGHT CAS 与 UNKNOWN Fail Closed。
- 冷启动 Refresh、前后台恢复、Single Flight、401 最多重试一次和 403 不刷新。
- `/api/iam/me` 权威 Mobile Access Context、Factory 白名单选择和无 Party 切换。
- 业务请求统一走 Auth Runtime，不接受页面传入 Access Token 或把 `X-Factory-Id` 当作凭证。
- 离线命令持久化完整用户、Session、Client、Factory、Party、Permission、幂等与关联归属，并在同步前逐项 Fail Closed。
- 登录切换前未同步命令确认门禁。

S12 仍需完成：

- 三仓库安全 E2E、敏感信息扫描、PR/CI 一致性与最终封板。

## 4. S11：Mobile Auth Runtime

实现：

- 固定 Client `mom-mobile-pda`。
- 只允许具备 Mobile Access 的 `INTERNAL` 用户。
- 系统浏览器 + Authorization Code + PKCE S256 + OIDC。
- 正式环境 HTTPS App Link。
- 可恢复的一次性 PKCE 事务。
- Access Token 只在内存。
- Refresh Token 通过 Android 安全存储 Adapter 持久化。
- Refresh Token 原子替换和不确定结果 Fail Safe。
- Single Flight Refresh。
- 冷启动、前后台和 Session 撤销恢复。
- `/api/iam/me`、Permissions、Factory 和 Mobile Access Context。
- 401、403、404、409、429、5xx 处理。
- 离线命令完整身份归属和同步门禁。
- H5 开发环境安全限制。

## 5. S11 验收

- 账号密码只在系统浏览器 IAM 页面输入。
- App Link 只能唤起预期 App/环境并校验 `state`、`nonce`、Client 和 Redirect URI。
- App 进程在浏览器登录期间被回收后仍可安全完成或终止事务。
- Access Token 不落盘。
- Refresh Token 不进入普通 Storage、日志、离线命令或诊断文件。
- 并发 401/前台恢复只产生一次 Refresh。
- 每个业务请求最多自动重试一次，403 不刷新。
- Refresh 本地替换不确定时不重复使用旧 Token。
- 冷启动能恢复有效 Session，撤销/到期后必须登录。
- `SUPPLIER`、`CUSTOMER` 或 Mobile Access 关闭用户被拒绝。
- 用户 B 不能自动同步用户 A 的命令。

## 6. S12：Mobile 安全 E2E

覆盖：

- 系统浏览器、PKCE、OIDC、App Link 域名和证书关联。
- Client/user_type/Mobile Access 隔离。
- Access Token 过期、Refresh Rotation、并发刷新、重放、撤销和 12 小时绝对期限。
- 冷启动、后台回前台、进程回收和网络不确定。
- Android 安全存储、退出、设备丢失和本地敏感信息扫描。
- `/api/iam/me`、Permission、Factory Scope 和 `X-Factory-Id`。
- 离线命令跨用户、跨 Session、跨 Factory、Permission 变化和 404 防枚举。
- H5 不持久化真实 Refresh Token。
- 真机和 Fake Adapter 的边界一致性。

## 7. 离线命令实施依赖

S11 必须扩展正式命令模型，至少包含：

- `userId`
- `sessionId` / `sid`
- `clientId`
- `factoryId`
- `partyType`
- `partyId`
- `requiredPermission`
- `createdAt`
- `idempotencyKey`
- `correlationId`
- `schemaVersion`

Token 不进入命令。同步时仍由 Gateway 和业务服务执行最终认证、授权、幂等与对象归属校验。
