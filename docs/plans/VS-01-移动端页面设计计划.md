# VS-01：移动端页面设计计划

## 1. 垂直切片目标

支持原料从供应商到货开始，经收货、待检、上架、生产领退料、成品入库和发运确认的现场 PDA 操作。

## 2. 设计批次

### Batch A：任务工作台与收货

页面：

- 首页任务工作台。
- 待收货任务列表。
- 送货单扫描。
- 容器与重量确认。
- 收货结果。

### Batch B：检验交接与上架

页面：

- 待检批次查询。
- 检验交接确认。
- 容器扫描。
- 库位扫描。
- 上架结果与异常。

### Batch C：生产领料与退料

页面：

- 工单任务列表。
- 工单扫描与需求明细。
- 物料/容器扫描。
- 数量确认。
- 退料原因与目标容器。

### Batch D：成品入库与发运

页面：

- 成品入库任务。
- 质量状态确认。
- 成品容器与库位双扫描。
- 发运单扫描。
- 装车确认。

### Batch E：离线与人工处理

页面：

- 离线命令列表。
- 命令详情。
- 同步进度。
- 冲突对比。
- 结果未知查询。
- 人工接管。

## 3. 每个页面必须提供

- 业务目标。
- 操作角色。
- 前置条件。
- 正常流程。
- 扫码来源和格式。
- 允许手工输入的字段。
- 正常竖屏原型。
- Loading、Empty、Offline、Conflict、Unknown、Hardware Error 原型。
- 页面状态矩阵。
- 组件映射。
- API、权限、工厂范围、幂等和离线映射。
- 验收场景。

## 4. 收货主流程

```text
打开待收货任务
→ 扫描送货单
→ 查询并校验送货信息
→ 扫描或生成容器
→ 输入重量
→ 查看确认摘要
→ 在线提交或离线入队
→ 展示业务单号 / 队列状态
```

## 5. 上架主流程

```text
扫描容器
→ 查询容器和质量状态
→ 扫描目标库位
→ 校验库区规则
→ 确认上架
→ 展示库存位置
```

## 6. 领料主流程

```text
扫描工单
→ 展示物料需求
→ 扫描物料批次或容器
→ 校验质量、库存和需求
→ 输入或确认数量
→ 提交领料
→ 展示累计结果
```

## 7. 离线设计要求

- 必须标记操作是否允许离线。
- 离线提交后显示命令编号和队列状态。
- 退出页面后可从离线队列找到命令。
- 网络恢复后不能静默隐藏同步失败。
- 冲突必须显示服务端当前状态和本地输入。
- 结果未知必须提供查询最终状态入口。

## 8. 原型验收

- 所有关键操作可在竖屏完成。
- 不依赖鼠标悬停。
- 主要按钮适合戴手套点击。
- 扫码状态和业务状态不混淆。
- 错误提示包含可执行恢复动作。
- 网络状态在页面上始终可见或易于发现。

## 9. 完成定义

VS-01 页面设计完成，不代表页面代码完成。设计完成至少要求：

- 页面清单冻结。
- 用户流程通过评审。
- 全部原型与状态图完成。
- API 和权限映射完成。
- 离线规则完成。
- 后端契约和错误码已确认。
