# 用户角色与设备场景

## 1. 仓库收货员

核心任务：

- 查看待收货任务。
- 扫描供应商送货单。
- 扫描容器并录入重量。
- 建立待检批次。
- 查看离线命令状态。

关注点：速度、重复扫描、重量校验、标签结果、网络中断。

## 2. 仓库上架员

核心任务：

- 扫描容器。
- 扫描目标库位。
- 处理库位不匹配、质量锁定和重复上架。

关注点：双扫描顺序、库区规则、操作反馈和人工接管。

## 3. 生产物料员

核心任务：

- 查看工单领料任务。
- 扫描工单、物料、容器和批次。
- 分次领料。
- 执行余料退回。

关注点：工单需求、批次状态、数量、超时后的结果未知。

## 4. 成品仓管员

核心任务：

- 查看成品入库任务。
- 扫描成品批次、容器和库位。
- 查看质量放行状态。
- 完成入库或转异常处理。

## 5. 发运操作员

核心任务：

- 扫描发运单。
- 扫描托盘、容器或批次。
- 校验客户、产品和数量。
- 完成装车确认。

## 6. 质量交接人员

核心任务：

- 扫描待检批次。
- 确认样品或批次交接。
- 查询检验与放行状态。

PDA 不承担复杂实验室检验录入。

## 7. 现场主管

核心任务：

- 查看失败、冲突和长期未同步命令。
- 查看设备和网络诊断。
- 执行人工接管、取消或重新分派。

## 8. 设备场景

### 8.1 相机扫码

H5 和通用 App 基线使用 `uni.scanCode`。

### 8.2 硬件扫描头

工业 PDA 可能通过广播、键盘模拟或厂商 SDK 返回扫码结果。必须通过 Scanner Adapter 接入，页面不得依赖厂商接口。

### 8.3 标签打印

V1 页面只展示打印请求和结果。打印协议与硬件选择需单独评估，并通过 Printer Adapter 接入。

### 8.4 振动和声音

不同结果建议使用不同反馈：

| 结果 | 建议反馈 |
|---|---|
| 扫码成功 | 短振动或短提示音 |
| 业务成功 | 明确绿色反馈和确认音 |
| 警告 | 黄色提示和双短振 |
| 失败 | 红色提示和长振动 |
| 冲突 | 红/橙色页面并要求人工确认 |

### 8.5 网络切换

设备可能在 Wi-Fi、蜂窝网络和断网之间切换。网络恢复后不能直接假定服务可用，应进行健康检查和受控同步。

## 9. 典型环境约束

- 戴手套。
- 单手操作。
- 屏幕较小。
- 环境噪声大。
- 光线复杂。
- 网络不稳定。
- 连续高频扫码。
- 设备电量和存储空间有限。

因此页面设计必须优先考虑大触控区域、少输入、短流程、强反馈和可恢复性。
