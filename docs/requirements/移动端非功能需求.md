# 移动端非功能需求

## 1. 可用性

- `NFR-MOB-USE-001`：主要操作按钮在常见工业 PDA 竖屏下应可单手点击。
- `NFR-MOB-USE-002`：核心作业不得依赖长文本输入。
- `NFR-MOB-USE-003`：扫码成功、业务成功、警告、冲突和失败应有可区分的视觉反馈。
- `NFR-MOB-USE-004`：戴手套场景下关键按钮触控区域不得过小。
- `NFR-MOB-USE-005`：任务流程应尽量保持一步一确认，避免复杂多层弹窗。

## 2. 弱网与离线

- `NFR-MOB-OFF-001`：网络状态变化应在 3 秒内反映到界面。
- `NFR-MOB-OFF-002`：离线命令在应用正常退出或重启后不得丢失。
- `NFR-MOB-OFF-003`：同步必须使用指数退避或受控重试，不得持续高频请求。
- `NFR-MOB-OFF-004`：网络恢复不代表命令可盲目执行，必须重新校验业务状态。
- `NFR-MOB-OFF-005`：不适合离线执行的高风险命令必须在断网时禁用并说明原因。

## 3. 幂等与一致性

- `NFR-MOB-IDEM-001`：每个写命令必须有稳定幂等键。
- `NFR-MOB-IDEM-002`：同一命令重试不得重新生成幂等键。
- `NFR-MOB-IDEM-003`：客户端幂等不替代服务端幂等。
- `NFR-MOB-IDEM-004`：超时后页面不得自动重复提交，应查询最终状态或进入人工处理。
- `NFR-MOB-IDEM-005`：冲突必须保留本地命令和服务端快照，禁止静默覆盖。

## 4. 性能

- `NFR-MOB-PERF-001`：应用首页骨架应在常见工业 PDA 上快速可见。
- `NFR-MOB-PERF-002`：扫码结果解析与页面反馈不应出现明显卡顿。
- `NFR-MOB-PERF-003`：离线队列列表应支持至少数百条命令的分页或虚拟化展示。
- `NFR-MOB-PERF-004`：本地存储读取不得阻塞页面长时间无反馈。
- `NFR-MOB-PERF-005`：图片、日志和缓存应有容量上限。

## 5. 安全

- `NFR-MOB-SEC-001`：Token、密码、私钥和签名材料不得写入日志。
- `NFR-MOB-SEC-002`：本地不得保存超出现场作业所需的完整业务数据。
- `NFR-MOB-SEC-003`：敏感本地数据应加密或避免落盘。
- `NFR-MOB-SEC-004`：应用切换账号或退出后必须清理会话相关缓存。
- `NFR-MOB-SEC-005`：发布包不得包含开发环境地址、测试账号或调试密钥。

## 6. 认证与权限

- `NFR-MOB-AUTH-001`：PDA 应使用 OAuth2.1/OIDC 兼容流程。
- `NFR-MOB-AUTH-002`：页面、操作按钮和数据范围必须受服务端权限约束。
- `NFR-MOB-AUTH-003`：本地权限只用于界面控制，不作为最终业务授权。
- `NFR-MOB-AUTH-004`：会话过期时应保存未提交输入并引导重新登录。

## 7. 设备兼容

- `NFR-MOB-DEV-001`：平台能力必须通过适配器暴露。
- `NFR-MOB-DEV-002`：真实 PDA SDK 接入不得要求重写业务页面。
- `NFR-MOB-DEV-003`：扫码权限、相机权限和存储权限必须可诊断。
- `NFR-MOB-DEV-004`：设备型号、系统版本和应用版本应能进入诊断信息。

## 8. 可观测性

- `NFR-MOB-OBS-001`：请求应携带 correlation ID。
- `NFR-MOB-OBS-002`：离线命令应保留业务类型、状态、尝试次数和最后错误。
- `NFR-MOB-OBS-003`：错误报告应包含应用版本、设备信息、页面和关联 ID。
- `NFR-MOB-OBS-004`：不得在高基数监控标签中使用用户、订单或批次编号。

## 9. 可维护性

- `NFR-MOB-MNT-001`：页面不得直接访问平台 API 或本地存储。
- `NFR-MOB-MNT-002`：API DTO 与页面 View Model 应在边界映射。
- `NFR-MOB-MNT-003`：离线状态机应有单元测试。
- `NFR-MOB-MNT-004`：关键设备行为应支持 Fake Adapter 测试。
- `NFR-MOB-MNT-005`：文档、代码和原型状态必须保持一致。

## 10. 发布与恢复

- `NFR-MOB-REL-001`：Android 包必须使用受控签名。
- `NFR-MOB-REL-002`：签名密钥不得进入 Git。
- `NFR-MOB-REL-003`：发布包应记录 Git SHA、版本和构建时间。
- `NFR-MOB-REL-004`：升级前后必须验证离线队列兼容性。
- `NFR-MOB-REL-005`：回滚不得导致未同步命令无法读取。
