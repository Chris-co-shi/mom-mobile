# Android 构建发布与签名

## 1. 产品目标

`mom-mobile` 的正式产品目标是 Android 工业 PDA。H5 仅用于 CI、原型评审和快速联调。

## 2. 当前状态

当前仓库具备 H5 构建，尚未完成：

- 目标 PDA 型号选择。
- 厂商扫描 SDK。
- Android 原生插件。
- 正式应用 ID。
- 签名密钥。
- 发布渠道和升级策略。

## 3. 构建方式

正式进入 Android Slice 前需决定：

- HBuilderX 云打包。
- uni-app CLI 本地打包。
- 原生工程集成。
- CI 中的可重复构建方案。

选择必须记录 ADR，并考虑供应链可追溯性。

## 4. 应用标识

每个环境应明确：

- App ID。
- Android Application ID。
- 显示名称。
- 图标。
- Deep Link 或自定义 Scheme。
- OAuth Redirect URI。

开发、测试和 prod-like 应避免误装或误连环境。

## 5. 版本

建议：

- `versionName`：语义版本。
- `versionCode`：单调递增整数。
- Git SHA。
- Build Time。
- Environment。

诊断页面应可查看版本信息。

## 6. 签名

- 签名密钥不得进入 Git。
- 密钥通过受控 Secret 或离线签名流程提供。
- 密钥访问需最小权限和审计。
- 需要安全备份和恢复方案。
- 测试签名与正式签名严格分离。

## 7. 权限

按最小权限申请：

- 相机。
- 网络。
- 必要存储能力。
- 厂商扫描服务权限。
- 振动。

未使用的敏感权限不得申请。

## 8. 设备兼容

发布前记录：

- PDA 型号。
- Android 版本。
- CPU 架构。
- 屏幕分辨率。
- 扫描器类型。
- 厂商 SDK 版本。
- WebView 版本要求。

## 9. 升级

升级需验证：

- 离线命令 Schema 迁移。
- 本地存储兼容。
- OAuth Redirect 不变或有迁移。
- 厂商 SDK 兼容。
- 页面路由兼容。

存在未同步命令时，升级前需提示并确保升级后可继续读取。

## 10. 回滚

- 保留上一稳定安装包。
- 回滚包与当前签名一致。
- 回滚后能够读取现有离线命令，或提供迁移工具。
- 数据格式不兼容时禁止直接覆盖安装。

## 11. 发布渠道

可选：

- 企业 MDM。
- 内部下载页。
- 设备厂商管理平台。
- 人工安装，仅用于早期 PoC。

正式发布应支持版本追踪和撤回。

## 12. 安全检查

- 包内无开发密钥。
- 无真实测试账号。
- 无生产 Token。
- Gateway 地址与环境一致。
- Source Map 和调试开关按环境控制。
- 第三方 SDK 来源和许可证已登记。

## 13. 发布验收

- 安装与启动成功。
- OAuth 登录成功。
- 扫描器工作。
- 网络切换正常。
- 离线队列可恢复。
- 升级和回滚验证完成。
- 版本和设备诊断信息正确。
