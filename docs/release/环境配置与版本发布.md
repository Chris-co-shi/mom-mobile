# 环境配置与版本发布

## 1. 环境

```text
local → dev → test → prod-like
```

各环境应使用独立 Gateway、OAuth Client、应用名称或明显环境标识。

## 2. 环境变量

当前示例：

```text
VITE_MOM_API_BASE_URL
```

后续可包括：

- OAuth Issuer。
- OAuth Client ID。
- Redirect URI。
- 日志或诊断开关。
- Feature Flag Endpoint。

浏览器可见或打包进入 App 的变量不得包含密钥。

## 3. 配置原则

- 不在页面中硬编码地址。
- 不提交真实 Token、密码、私钥或签名材料。
- 生产配置不使用开发默认值。
- 环境差异可审查。
- 构建产物可追溯到 Git SHA。

## 4. Feature Flag

高风险功能可通过受控开关发布，例如：

- 自动同步 Worker。
- 连续扫码。
- 厂商扫描 SDK。
- 标签打印。
- 新冲突处理流程。

开关必须有默认值、环境范围和回滚方式。

## 5. 发布产物

每个产物记录：

- 版本名称。
- 版本号。
- Git SHA。
- 构建时间。
- 目标环境。
- 签名标识。
- 目标设备型号。
- 依赖和 SDK 版本。

## 6. 发布步骤

```text
合并代码
→ CI 检查
→ 生成候选包
→ test 真机验证
→ 离线/升级/回滚验证
→ 生成 prod-like 包
→ 发布说明
→ 灰度安装
→ 观察
→ 扩大范围
```

## 7. 发布说明

至少包含：

- 新增功能。
- 修复问题。
- 已知限制。
- 是否修改离线命令 Schema。
- 是否要求重新登录。
- 是否修改设备权限。
- 是否修改扫描 SDK。
- 升级与回滚说明。

## 8. 灰度

建议先选择：

- 少量测试设备。
- 单个仓库或班组。
- 明确设备型号。
- 可随时回滚的用户。

重点观察：

- 启动失败。
- 扫码失败。
- 登录失败。
- 离线队列增长。
- 冲突率。
- 网络重试。
- 本地存储和电量。

## 9. 回滚条件

- 核心任务无法完成。
- 离线命令不可读取。
- 扫描头失效。
- 登录或 Gateway 大面积失败。
- 数据重复或状态错误风险。
- 应用频繁崩溃。

## 10. 验收

- 不同环境不会误连。
- 包版本可在应用内查看。
- 发布包可追溯。
- 灰度与回滚路径明确。
- 升级前后离线队列兼容。
